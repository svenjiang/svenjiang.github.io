交互模式：精简输出 + 关键节点确认
默认模式。Agent 流式输出思考和操作摘要，在需要用户决策的地方暂停等待输入。适合日常开发使用。
claude> fix the failing tests in src/auth/
● Reading test output...
● Found 2 failures in token.test.ts
● Edit src/auth/token.ts
diff preview
+ if (!token) throw new AppError('Missing token', 401)
- if (!token) return null
Apply this change? [y/n] y
✓ 2 tests passing
交互模式的核心体验：用户能看到 Agent 在"想什么"，并在每次文件变更前看到完整的 diff。确认粒度精确到每一次写入操作。
详细模式：完整 Thought + Tool 调用链
加上 --verbose 标志后，输出每一轮 ReAct 循环的完整内容——包括 Thought 推理过程、工具参数、token 消耗。适合调试和学习 Agent 行为。
--verbose -v
claude --verbose> fix the failing tests
── Turn 1 ──────────────────────────
Thought: I need to first understand what tests are failing.
  Will run the test suite to get the error output.
Action: Bash
  command: "npm test -- --reporter=verbose 2>&1 | tail -40"
  timeout: 30000
Observation: [stdout 847 chars]
  FAIL src/auth/token.test.ts
  ✗ should return 401 for missing token
── Turn 2 ──────────────────────────
Thought: Error is in token.ts line 24. The function
  returns null instead of throwing. Fix with Edit.
Action: Edit
  file: "src/auth/token.ts"
  old: "if (!token) return null"
  new: "if (!token) throw new AppError('Missing token', 401)"
── Cost ────────────────────────────
Input: 2,841 tokens · Output: 312 tokens · $0.0038
--verbose 是理解 Agent 决策的最佳工具。当 Agent 做出了你不理解的操作时，加上这个标志重跑，可以看到完整的推理链。也是优化 CLAUDE.md 内容的重要参考。
Headless / CI 模式：非交互，结构化输出
使用 --print 标志（或 -p）进入非交互模式。Agent 直接执行完整任务，将最终输出打印到 stdout，不等待任何用户输入。适合 CI/CD 流水线。
--print -p --output-format json --dangerously-skip-permissions
# GitHub Actions workflow
- name: Claude Code Review
  run: |
    claude -p \
      --output-format json \
      "Review the diff in $PR_DIFF,
       list any security issues" \
      | jq '.result'
# stdout →
{
  "result": "No security issues found. The diff...",
  "cost": { "input_tokens": 4102, "output_tokens": 89 },
  "session_id": "sess_01XYZ..."
}
CI 模式的关键：--output-format json 让下游工具能程序化处理结果；--dangerously-skip-permissions 只在 CI 隔离环境里使用，配合 Docker 容器确保安全边界。
