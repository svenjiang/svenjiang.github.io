stdio 传输：本地进程 MCP Server
MCP server 以子进程方式启动，通过标准输入输出通信。延迟低，适合本地工具。配置写入 .claude/claude.json 或 ~/.claude/claude.json。
stdio本地子进程，低延迟
// .claude/claude.json（项目级，随 git 提交）
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/home/user/projects/myapp"
      ]
    },
    "sqlite": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-sqlite"],
      "env": {
        "SQLITE_DB_PATH": "./data/app.db"
      }
    },
    "internal-api": {
      "command": "node",
      "args": ["./tools/mcp-server/dist/index.js"],
      "env": {
        "API_TOKEN": "${INTERNAL_API_TOKEN}"
      }
    }
  }
}
环境变量用 ${VAR_NAME} 语法引用系统环境变量，避免把敏感 token 硬编码进配置文件。
SSE 传输：远程 MCP Server
MCP server 作为独立 HTTP 服务运行，通过 Server-Sent Events 通信。适合团队共享的云端工具，或需要持久连接的远程服务。
SSE / HTTP远程服务，支持团队共享
// ~/.claude/claude.json（用户全局，个人工具）
{
  "mcpServers": {
    "github": {
      "type": "url",
      "url": "https://api.githubcopilot.com/mcp/v1",
      "headers": {
        "Authorization": "Bearer ${GITHUB_TOKEN}"
      }
    },
    "linear": {
      "type": "url",
      "url": "https://mcp.linear.app/sse",
      "headers": {
        "Authorization": "Bearer ${LINEAR_API_KEY}"
      }
    },
    "team-tools": {
      "type": "url",
      "url": "https://tools.company.internal/mcp/v1"
    }
  }
}
远程 MCP server 适合团队场景：部署一个 server，全团队共享，无需每人本地安装和维护。结合企业级配置可以强制所有成员接入合规审计工具。
调用示例：GitHub MCP Server
任务：查找本周新增的 open issues，按优先级整理成摘要报告。Agent 通过 GitHub MCP Server 完成，全程无需 Bash 调用 gh CLI。
Thought 需要查本周 open issues。GitHub MCP Server 提供了 list_issues 工具，可以按创建时间过滤。
Action mcp:github → list_issues
参数：{"state": "open", "since": "2025-01-20", "per_page": 50}
Observation 返回 23 个 issues，包含标题、标签、assignee、创建时间。其中标记 priority:high 的有 5 个。
Thought 高优先级 issue 需要补充评论数和最近活动，调用 get_issue 获取详情。
Action mcp:github → get_issue × 5（并行，通过 Task 工具）
获取每个高优先级 issue 的完整详情。
完成 整理成 Markdown 报告：高优先级 5 条（含责任人和截止时间）+ 普通 issue 18 条汇总。
MCP 工具调用与内置工具调用在 Agent 视角完全一致——都是 Action → Observation 的循环。Agent 不需要知道工具是内置的还是来自 MCP Server。
调用示例：数据库 MCP Server
任务：分析过去 30 天用户注册趋势，找出异常下降的日期。Agent 通过 PostgreSQL MCP Server 直接查询，无需用户提供连接字符串。
Thought 先了解数据库 schema，确认用户表的结构和时间字段名称，再写查询。
Action mcp:postgres → list_tables
获取所有表名和列定义。
Observation 找到 users 表，注册时间字段为 created_at（timestamp with time zone）。
Action mcp:postgres → execute_query
SELECT DATE(created_at), COUNT(*) FROM users WHERE created_at > NOW() - INTERVAL '30 days' GROUP BY 1 ORDER BY 1
Observation 返回 30 行数据。发现 2025-01-15 注册量为 12，前后两天均在 150-200 之间，异常明显。
完成 输出分析报告，标注 1 月 15 日异常（降幅 92%），建议检查当日部署记录和错误日志。
数据库 MCP Server 通常只开放 SELECT 权限，写操作需要额外配置和确认。这是"最小必要权限"在 MCP 层的具体体现——工具只暴露它实际需要的能力。
