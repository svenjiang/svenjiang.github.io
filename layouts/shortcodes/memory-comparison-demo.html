压缩 自动触发 compaction：Turn 1-18 被摘要替换，保留 Turn 19-20 原文
Turn 21 基于摘要 + 最新历史继续执行，部分中间细节已丢失
Bash 工具的 stdout 是上下文消耗大户——一次完整的 npm test 输出可能超过 2,000 tokens。对于长任务，建议用 | tail -30 截断输出，只保留最相关的部分。
压缩前 vs 压缩后：信息损失在哪里
压缩是"有损摘要"——保留目标和结论，丢弃中间过程。了解哪些信息容易丢失，才能提前做好保护。

恢复会话：--continue 与 --resume
Claude Code 将对话历史序列化到磁盘，支持跨进程恢复。适合跨天的长任务，或意外中断后的继续。
--continue
自动接续最近一次未完成会话
--resume
列出历史会话，手动选择恢复
存储位置
~/.claude/sessions/
包含内容
完整对话历史（含压缩摘要）
Day 1 完成认证模块，Ctrl+C 中断会话。历史自动序列化到 ~/.claude/sessions/sess_01XYZ.json
Day 2 claude --continue — 自动加载上次会话，从中断处继续
注意 恢复的历史是"冷"的：模型重新处理这段历史时，对细节的激活程度不如当时执行时高
建议 每天结束时，让 Agent 写一段"当前状态摘要"到 CLAUDE.md，作为次日恢复的锚点
# 最佳实践：每日收尾时执行
claude> 今天的工作到此结束。
        请把当前的进度状态、明天需要继续的任务、
        以及今天发现的重要信息写入 CLAUDE.md。
        明天我会用 --continue 恢复。
CLAUDE.md 是比 --continue 更可靠的跨日记忆载体。历史文件包含所有原始对话，但 CLAUDE.md 包含的是经过整理的"精华"——后者在下次会话中的利用效率更高。
